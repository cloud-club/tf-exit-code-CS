## [COMMON-041] 프로세스 주소 공간이 어떻게 구성되는지 설명해 주세요.

답변:

프로세스 주소 공간은 프로세스가 실행되기 위해 운영체제가 할당하는 공간으로, Code 영역, Data 영역, Heap 영역, Stack 영역으로 구성됩니다.

먼저 Code 영역은 프로그램의 실행 코드가 저장되는 영역이고, Data 영역은 전역 변수, 정적 변수와 같이 프로그램 시작 시 초기화된 값이 저장되는 영역입니다.

Heap 영역은 프로세스 실행 중에 동적으로 할당되는 메모리 공간이고, Stack 영역은 프로세스 내 함수 호출 시 생성되는 지역 변수, 매개 변수 등의 값이 저장되는 영역입니다.

기본적으로 프로세스 Code 영역, Data 영역, Heap 영역은 낮은 주소부터 공간이 할당되고, Stack 영역은 높은 주소부터 공간이 할당되어 낮은 주소에 점차 공간이 할당됩니다.

## [COMMON-043] Stack과 Heap의 차이에 대해 설명해 주세요.

답변:

Stack 영역은 함수가 호출될 때 지역 변수, 매개변수 등을 저장하기 위해 자동으로 할당되고, 함수가 종료되면 해제되는 영역입니다.

이렇게 컴파일러가 할당과 해제를 알아서 관리해주기 때문에 별도로 신경 쓸 필요가 없고, 자료구조 특성상 단순히 스택 포인터를 이동시키는 방식으로 동작하기 때문에 할당 속도도 빠릅니다. 다만 프로세스마다 정해진 크기 이상으로는 사용할 수 없어 할당 크기 이상으로 공간을 활용하면 스택 오버플로우가 발생할 수 있습니다.

반면 Heap 영역은 프로그래머가 `malloc`이나 `new` 같은 함수를 이용해 명시적으로 할당하고, `free` 등으로 직접 해제해야 하는 영역입니다. 크기를 동적으로 늘리거나 줄일 수 있어서 Stack보다 훨씬 유연하지만, 빈 공간을 탐색하는 속도가 상대적으로 느리고, 할당 공간을 해제하는 등의 작업을 수동적으로 해야 합니다.

## [COMMON-044] Stack Overflow가 발생하는 이유를 설명해 주세요.

답변:

Stack 영역은 프로세스마다 정해진 크기만큼만 할당되는 공간입니다. 프로스세 내에서 함수를 호출할 떄마다 스택에 프레임이 쌓이는데, 정해진 크기 이상으로 프레임이 공간을 차지하게 되면 Stack Overflow가 발생합니다. 주로 재귀 함수를 호출했지만 함수의 종료 코드가 없어 함수가 무한적으로 호출되거나, Stack에 저장되는 값의 크기가 지나치게 클 경우 발생합니다.

## [COMMON-045] Memory Leak이 무엇이고, 어떤 상황에서 발생할 수 있는지 설명해 주세요.

답변:

Memory Leak란, 메모리 누수 현상으로 동적으로 할당되는 Heap 메모리 공간을 더이상 사용하지 않음에도 할당을 해제하지 않아서 프로세스의 사용 가능한 메모리가 줄어드는 현상을 의미합니다. 이러한 현상이 지속되면 메모리 자원이 고갈되어 성능이 저하될 수 있고, 프로세스가 종료될 수 있습니다.

주로 C/C++ 프로그래밍에서 `malloc()` 과 같은 함수를 호출하여 공간을 할당 받고, 할당받은 공간을 `free()`로 해제하지 않는 경우에 발생할 수 있습니다.

## [COMMON-046] 가상 메모리가 무엇이고, 왜 필요한지 설명해 주세요.

답변:

가상 메모리는 프로세스의 데이터가 실제 메모리의 어디에 배치되어있는지 상관없이, 독립적이고 연속적인 주소 공간을 사용하는 것처럼 동작하게 하는 메모리 관리 기법입니다.

실제로 메모리에는 여러 프로세스의 데이터가 서로 흩어져서 배치될 수 있기 때문에 실제 주소를 활용하게 되면 다른 프로세스의 영역을 침범할 수도 있고, 흩어진 데이터의 위치를 찾는 과정에서 성능이 지연될 수 있습니다. 이를 위해 CPU가 프로세스의 주소 공간을 참조할 때, MMU에 의해 논리적인 주소를 실제 물리 주소로 변환하는 방식을 사용함으로써 흩어져있는 데이터를 가상의 독립적이고 연속적인 공간으로 활용할 수 있습니다.

## [COMMON-047] 페이징이 무엇이고, 페이지와 프레임의 차이를 설명해 주세요.

답변:

페이징은 프로세스를 연속적으로 메모리에 할당함으로써 발생할 수 있는 단편화 문제를 해결하기 위한 분산 메모리 할당 기법입니다.

페이징은 가상 주소 공간과 물리 주소 공간을 동일한 크기로 나눠, 프로세스를 비연속적으로 메모리에 할당할 수 있도록 하는 기법입니다.

여기서 페이지는 가상 주소 공간을 나누는 단위이고, 프레임은 물리 주소 공간을 나누는 단위입니다. 각 프로세스마다 페이지 테이블을 관리하고 있고, 이를 통해 각 페이지가 어떤 프레임과 매핑되어있는지를 추적함으로써 프로세스 정보가 비연속적으로 할당되어있음에도 논리적으로 연속적인 메모리 공간을 활용할 수 있습니다.

## [COMMON-048] Page Fault가 발생했을 때 처리 과정을 설명해 주세요. ⭐

답변:

Page Fault는 페이지가 현재 물리 메모리에 적재되어 있지 않은 것을 의미합니다.

먼저 메모리에서 비어있는 프레임을 찾고, 빈 프레임이 없다면 페이지 교체 알고리즘을 활용하여 가져올 페이지를 적재할 공간을 확보합니다.

필요한 페이지를 디스크에서 프레임으로 읽어오고, 페이지 테이블을 갱신합니다.

## [COMMON-049] TLB가 무엇이고, TLB를 사용하면 왜 주소 변환이 빨라지는지 설명해 주세요. ⭐

답변:

TLB는 가상 주소를 물리 주소로 변환한 결과를 저장해두는 MMU 내부의 캐시 메모리입니다.

페이징 기법에서는 원래 가상 주소를 물리 주소로 변환하기 위해 페이지 테이블에서 물리 주소 값을 조회해야 합니다.

하지만 페이지 테이블도 메모리에 저장되어 있으므로, 페이지 테이블 조회를 위한 메모리 접근 1회, 조회된 물리 주소 조회에 접근하기 위한 메모리 접근 1회, 총 두 번의 메모리 접근이 필요합니다.

TLB는 최근에 사용된 가상주소와 물리 주소 매핑을 캐싱해두기 때문에, 먼저 TLB를 확인하여 매핑 정보가 있으면 물리 주소를 얻어 메모리 접근 1회로 끝낼 수 있기 때문에 주소 변환이 빨라집니다.

## [COMMON-050] 내부 단편화와 외부 단편화의 차이에 대해 설명해 주세요.

답변:

내부 단편화는 메모리 공간을 고정된 크기로 할당할 때, 프로세스가 실제로 사용하는 크기가 할당된 메모리 크기보다 작아 해당 블록 내부의 메모리 공간이 낭비되는 것을 의미합니다.

외부 단편화는 메모리를 가변 크기로 동적으로 할당할 때, 여러 프로세스의 할당, 해제가 반복되면서 사용 가능한 빈 메모리 공간이 생기지만, 하나의 프로세스가 사용할만큼 크기가 충분하지 않아 공간이 낭비되는 것을 의미합니다.

## [COMMON-051] 동시성과 병렬성의 차이에 대해 설명해 주세요.

답변:

동시성은 하나의 CPU 코어에서 Context Switching을 통해 여러 작업을 짧은 시간에 번갈아가며 실행함으로써 동시에 실행되는 것처럼 보이게 하는 개념입니다.

병렬성은 실제로 여러 개의 CPU 코어에서 여러 작업이 동시에 실행되는 것입니다.

동시성은 물리적으로 동시에 실행되는 것은 아니지만, 동시에 실행되는 것처럼 보이게 하는 것이고, 병렬성은 실제로 물리적으로 동시에 실행되는 것입니다.

## [COMMON-052] Race Condition이 무엇이고, 어떤 상황에서 발생하는지 설명해 주세요.

답변:

Race Condition이란, 여러 프로세스가 공유 자원에 동시에 접근할 때, 실행 순서에 따라 결과가 달라지는 상황을 의미합니다.

예를 들어, 2개의 프로세스가 하나의 정수형 변수에 +1을 하는 경우, 동시에 기존 변수 값을 읽고, 연산하여 값을 쓰게 될 경우 2개의 프로세스가 +1을 하기 때문에 +2가 되어야 하지만, 실제로는 +1이 되는 상황을 의미합니다.

주로 여러 스레드나 프로세스가 공유 자원에 동시에 접근할 때, 이 임계 영역에 대한 상호배제가 보장되지 않을 때 발생합니다.

## [COMMON-053] Critical Section이 무엇이고, 임계 구역 문제를 해결하기 위한 조건을 설명해 주세요. ⭐

답변:

Critical Section이란 여러 프로세스나 스레드가 공유 자원에 접근하는 코드 영역으로, 한 번에 하나의 프로세스나 스레드가 접근해야 하는 영역을 의미합니다.

임계 구역 문제를 해결하기 위해서는 상호 배제, 진행, 유한 대기 조건을 만족해야 합니다.

먼저 상호 배제는 한 프로세스나 스레드가 임계 영역에서 실행 중인 경우, 다른 프로세스는 해당 임계 영역에 진입할 수 없어야 하는 것을 의미합니다.

진행 조건은 여러 프로세스나 스레드가 임계 영역에 접근하고자 할 때, 어느 프로세스가 접근할지 결졍되는 과정이 지연되면 안되는 것을 의미합니다.

마지막으로 유한 대기 조건이란, 여러 프로세스나 스레드가 임계 영역에서 번갈아가며 실행 될 때, 모두 무한정 기다리지 않아야 하는 것을 의미합니다.

## [COMMON-054] Mutex와 Semaphore의 차이에 대해 설명해 주세요. ⭐

답변:

Mutex란, Mutual Exclusion 하나의 자원에 오직 하나의 스레드만 접근할 수 있도록 하는 것을 의미합니다. 따라서 공유 자원을 사용 중인지, 사용 중이 아닌지를 추적하고, 자원을 사용 중 스레드만이 자원 사용을 해제해야 하는 소유권 개념이 있습니다.

Semaphore란, 카운터를 통해 동시에 접근 가능한 자원의 개수를 제어하는 것을 의미합니다. 카운터가 1이라면 하나의 스레드만 공유 자원에 접근할 수 있지만, 카운터가 n이라면 n개의 스레드가 공유 자원에 접근할 수 있습니다.

Mutex는 소유권 개념이 있고 하나의 자원에 하나의 스레드만 접근하는 반면, Semaphore는 여러 스레드가 제한된 개수만큼 공유자원에 동시에 접근할 수 있도록 한다는 차이점이 있습니다.

## [COMMON-055] Deadlock이 무엇이고, 발생 조건 4가지를 설명해 주세요.

답변:

Deadlock이란 두 개 이상의 프로세스(또는 스레드)가 서로 상대방이 접근한 자원을 무한정으로 기다리는 상태를 의미합니다.

Deadlock은 상호와 배제, 점유와 대기, 비선점, 순환 대기 조건이 모두 성립해야 발생합니다.

먼저 상호 배제는 한 번에 하나의 프로세스만 사용할 수 있는 자원이 있어야 한다는 것을 의미합니다.

점유와 대기는 프로세스가 자원을 점유한 상태에서, 다른 프로세스가 해당 자원에 점유하기 위해 대기하는 것을 의미합니다.

비선점은 다른 프로세스에게 할당된 자원은 그 프로세스가 할당을 해제하기 전까지 다른 프로세스가 점유할 수 없다는 것을 의미합니다.

순환 대기는 프로세스들이 자원을 기다리고, 점유하는 관계가 순환적이라는 것을 의미합니다.

따라서 이 4가지 조건 중 하나라도 성립하지 않게 하면 Deadlock을 예방할 수 있습니다.

## [COMMON-056] Deadlock을 예방하거나 회피하는 방법에 대해 설명해 주세요. ⭐

답변:

Deadlock을 예방하는 방법은 예방, 회피, 탐지 후 회복, 무시가 있습니다.

먼저 예방은 Deadlock의 4가지 필요조건 중 하나를 원천적으로 성립하지 못하게 막는 방법입니다.

회피 프로세스가 안전한 상태를 유지하는 경우에만 공유 자원 접근을 허용하는 방법입니다.

탐지 및 회복은 Deadlock 발생을 막지 않고 주기적으로 Deadlock을 탐지한 뒤, 발견되면 프로세스를 종료하거나 자원을 선점하여 해결하는 방식입니다.

무시는 Deadlock이 자주 발생하지 않을 것 같은 경우 별도의 조치를 하지 않고, 발생 시 프로세스 재시작 등으로 대응하는 방식입니다.

## [COMMON-058] Hash Table이 무엇이고, 해시 충돌을 해결하는 방법을 설명해 주세요.

답변:

Hash Table은 키를 해시 함수를 통해 얻은 값을 인덱스로 사용해 데이터를 저장하는 자료구조입니다. 해시 함수를 통해 인덱스를 바로 계산할 수 있기 때문에 평균적으로 삽입, 삭제, 탐색 등의 작업을 O(1)에 수행할 수 있습니다.

하지만 서로 다른 키가 같은 해시 값으로 매핑되는 해시 충돌이 발생할 수 있는데, 이는 체이닝, 개방 주소법으로 해결할 수 있습니다.

먼저 체이닝은 같은 인덱스에 해당하는 데이터들을 연결 리스트로 연결하여 저장하는 방식입니다. 충돌이 발생해도 이를 중복으로 취급하여 허용하는 방식이라고 할 수 있습니다. 하지만 인덱스의 중복이 많이 발생할 수록 그만큼 탐색 시간이 늘어날 수 있다는 단점이 있습니다.

개방 주소법은 충돌이 발생하면 다른 빈 인덱스에 값을 저장하는 방식입니다.

## [COMMON-059] Stack, Queue, Deque의 차이와 사용 사례를 설명해 주세요. ⭐

답변:

먼저 Stack은 Last In First Out 구조로, 가장 나중에 들어간 데이터가 가장 먼저 나오는 자료구조입니다. 한쪽 끝에서 스택 포인터를 활용하여 push와 pop을 통해 데이터를 추가 및 삭제하는 자료구조입니다. 대표적으로 프로세스 주소 공간에서 함수가 호출되는 공간에 사용됩니다.

Queue는 First In First Out 구조로, 가장 먼저 들어간 데이터가 가장 먼저 나오는 자료구조입니다. 데이터의 추가는 뒤에서, 삭제는 앞에서 발생합니다. 대표적으로 프로세스의 작업을 스케줄링할 때 큐가 사용됩니다.

Deque은 양쪽 끝 모두에서 삽입과 삭제가 가능한 자료구조로, Stack과 Queue의 기능을 모두 포함할 수 있는 확장된 구조입니다.

## [COMMON-060] Tree, Binary Tree, Binary Search Tree의 차이에 대해 설명해 주세요.

답변:

먼저 Tree는 노드와 간선으로 이루어진 비선형 계층 구조로, 하나의 루트 노드에서 시작해 자식 노드로 뻗어나가며 사이클이 없는 계층형 구조입니다.

Binary Tree는 Tree 중에서 각 노드가 최대 2개의 자식만을 가질 수 있는 자료구조입니다.

Binary Search Tree는 왼쪽 서브트리의 모든 노드 값은 부모보다 작고, 오른쪽 서브트리의 모든 노드 값은 부모보다 크다는 규칙을 만족하는 Binary Tree입니다.
