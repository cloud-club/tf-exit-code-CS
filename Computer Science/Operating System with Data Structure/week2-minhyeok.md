## [COMMON-042] Code, Data, Heap, Stack 영역의 역할을 설명해 주세요.

답변: code -> 말 그대로 코드를 저장함(IR이 레퍼런스하는 영역) Data -> 플레인 text와 같은 리소스들을 저장하는 역역, heap -> 동적 변수 할당 등을 위한 공간, Stack -> 스택 프레임 안에서 함수의 흐름, 지역변수 등을 저장하는 공간

## [COMMON-043] Stack과 Heap의 차이에 대해 설명해 주세요.

## [COMMON-044] Stack Overflow가 발생하는 이유를 설명해 주세요.

답변: 할당된 스택의 변수 범위를 초과해서 값을 할당을 할때 발생. 스택 영역의 허용되지 않는 다른 메모리 값을 변경함.

답변: 메모리 할당 후에 release를 하지 않은 경우. 동적 할당에서 malloc 이후 free하지 않을 때 발생

## [COMMON-046] 가상 메모리가 무엇이고, 왜 필요한지 설명해 주세요.

답변: 주메모리는 크기가 작아서 프로세스의 모든 데이터를 올리지 못하기 위해 메모리 - 디스크 간의 가상 주소를 만들고 필요할때마다 디스크에서 메모리로 올려야함. 이때 필요한 개념이 가상 메모리 개념

참고 자료:

## [COMMON-047] 페이징이 무엇이고, 페이지와 프레임의 차이를 설명해 주세요.

답변: 물리 메모리를 고정 크기 단위로 나눠 관리하는 기법. 가상 주소 공간을 나눈 단위가 페이지, 물리 메모리를 나눈 단위가 프레임이고 페이지 테이블이 둘을 매핑함. (디스크->메모리로 올리는 건 디맨드 페이징)

참고 자료:

## [COMMON-048] Page Fault가 발생했을 때 처리 과정을 설명해 주세요.

답변: 접근한 페이지가 메모리에 없을 때 나는 예외. 커널이 디스크에서 해당 페이지를 빈 프레임에 로드하고 페이지 테이블 갱신 후 명령어를 재실행함. 빈 프레임이 없으면 교체 알고리즘으로 희생 페이지를 내보냄.

참고 자료:

## [COMMON-049] TLB가 무엇이고, TLB를 사용하면 왜 주소 변환이 빨라지는지 설명해 주세요.

답변: 프로세스를 사용하기 위해서 2번의 메모리 접근이 필요한데(페이지 넘버 -> 프레임 넘버, 프레임 넘버 -> 물리 주소) 이를 캐시를 이용해서 메모리 접근을 빠르게 만든 것

## [COMMON-050] 내부 단편화와 외부 단편화의 차이에 대해 설명해 주세요.

답변: 내부 - 고정 크기 페이징에서 마지막 페이지에 다 못 채운 자투리 공간이 생기는 것, 외부 - 세그멘테이션(가변 분할)에서 할당/해제를 반복하다 사이사이 어중간한 틈이 생겨, 총 공간은 충분한데 연속 자리가 없어 못 쓰는 것

## [COMMON-051] 동시성과 병렬성의 차이에 대해 설명해 주세요.

답변: 동시성 -> 여러 작업을 번갈아 처리해 동시에 진행되는 것처럼 보이게 함(단일 코어도 가능) / 병렬성 -> 여러 코어에서 실제로 동시에 실행. 동시성은 구조의 문제, 병렬성은 실행의 문제.

참고 자료:

---

## [COMMON-052] Race Condition이 무엇이고, 어떤 상황에서 발생하는지 설명해 주세요.

답변: 둘 이상의 스레드가 공유 자원에 동시 접근하고 하나 이상이 쓰기를 할 때, 실행 순서에 따라 결과가 달라지는 것. 동기화 없이 read-modify-write(예: count++)를 하면 발생.

참고 자료:

---

## [COMMON-053] Critical Section이 무엇이고, 임계 구역 문제를 해결하기 위한 조건을 설명해 주세요.

답변: 공유 자원에 접근하는 코드 구역. 해결 조건 3가지 -> 상호 배제(한 번에 하나만 진입), 진행(비어있으면 진입 결정 지연 없음), 한정 대기(무한정 대기 없음).

참고 자료:

---

## [COMMON-054] Mutex와 Semaphore의 차이에 대해 설명해 주세요.

답변: Mutex -> 하나의 자원에 대한 잠금, 소유권 있음(잠근 스레드만 해제 가능) / Semaphore -> 카운터로 N개 자원을 관리, 소유권 없음. 뮤텍스는 상호 배제용, 세마포어는 자원 개수 제어/시그널링용.

참고 자료:

---

## [COMMON-055] Deadlock이 무엇이고, 발생 조건 4가지를 설명해 주세요.

답변: 둘 이상의 프로세스가 서로가 가진 자원을 기다리며 무한 대기하는 상태. 4조건 -> 상호 배제, 점유와 대기(hold and wait), 비선점, 순환 대기. 네 개가 동시에 성립할 때 발생.

참고 자료:

---

## [COMMON-056] Deadlock을 예방하거나 회피하는 방법에 대해 설명해 주세요.

답변: 예방 -> 4조건 중 하나를 깬다(자원 요청 순서를 정해 순환 대기 차단 등) / 회피 -> 은행원 알고리즘으로 안전 상태만 유지. 그 외 탐지 후 복구, 무시(타조 알고리즘)도 있음.

참고 자료:

---

## [COMMON-057] Thread Safe하다는 것은 어떤 의미인지 설명해 주세요.

답변: 여러 스레드가 동시에 접근해도 결과의 정확성이 깨지지 않는 것. 동기화(락), 불변 객체, 공유 상태 없이 지역 변수만 사용 등으로 달성.

참고 자료:

---

## [COMMON-058] Hash Table이 무엇이고, 해시 충돌을 해결하는 방법을 설명해 주세요.

답변: 키를 해시 함수로 인덱스로 바꿔 평균 O(1)로 조회하는 자료구조. 충돌 해결 -> 체이닝(버킷을 연결 리스트로), 개방 주소법(선형/이차 탐사, 이중 해싱).

참고 자료:

---

## [COMMON-059] Stack, Queue, Deque의 차이와 사용 사례를 설명해 주세요.

답변: Stack -> LIFO(함수 호출, undo, DFS) / Queue -> FIFO(작업 큐, BFS) / Deque -> 양쪽 끝 삽입·삭제 가능(슬라이딩 윈도우 최댓값).

참고 자료:

## [COMMON-060] Tree, Binary Tree, Binary Search Tree의 차이에 대해 설명해 주세요.

답변: Tree -> 사이클 없는 계층 구조 / Binary Tree -> 자식이 최대 2개인 트리 / BST -> 왼쪽 < 부모 < 오른쪽 정렬 조건이 있어 탐색이 평균 O(log n)(단, 균형이 깨지면 O(n)).
