## [COMMON-042] Code, Data, Heap, Stack 영역의 역할을 설명해 주세요.

답변: code 영역은 프로그램의 실행 명령어가 저장되는 영역, 일반적으로 읽기전용. data영역은 전역변수, 정적 변수가 저장되는 영역으로 초기화된 변수/초기화되지 않은 변수 영역으로 나뉠 수 있음. heap은 실행 중 동적으로 할당되는 메모리, stack은 함수 호출 정보, 지역변수 저장. 함수 호출 시 생성되고 함수가 종료되면 자동 제거.

## [COMMON-043] Stack과 Heap의 차이에 대해 설명해 주세요.

답변: stack은 함수 호출 과정에서 생성되는 지역 변수, 매개변수, 반환 주소 등을 저장. 메모리 할당, 해제가 자동으로 이뤄지고 속도가 빠르지만 크기가 제한적. heap은 실행 중 필요한 데이터를 동적 할당하는 영역. stack보다 큰 데이터를 저장할 수 있지만 메모리 할당과 해제비용이 더 큼. 즉 stack은 주로 짧은 생명주기의 데이터를 저장하고, heap은 함수 호출이 끝난 뒤에도 유지해야 하는 데이터를 저장할 때 사용

## [COMMON-045] Memory Leak이 무엇이고, 어떤 상황에서 발생할 수 있는지 설명해 주세요.

답변: 프로그램이 사용한 메모리를 더 이상 필요로 하지 않는데도 해제하지 못해 계속 점유하는 현상. c, c++에서는 malloc, new 로 할당한 메모리를 free나 delete로 해제하지 않았을 때 발생. 가비지 컬렉션을 사용하는 언어에서도 사용하지 않는 객체를 전역변수, 컬렉션, 이벤트 리스너 등이 계속 참조하면 가비지 컬렉터가 해당 객체를 제거하지 못해 메모리 누수 발생 가능

## [COMMON-046] 가상 메모리가 무엇이고, 왜 필요한지 설명해 주세요.

답변: 각 프로세스에 실제 물리메모리와 분리된 독립적 주소공간을 제공하는 메모리 관리 기법. 프로세스가 실제 RAM보다 큰 주소 공간을 사용 가능, 필요한 부분만 메모리에 올려 효율적 실행 가능. 프로그램마다 주소공간 분리하여 다른 프로세스 메모리를 침범핮 못하도록 보호 가능

## [COMMON-047] 페이징이 무엇이고, 페이지와 프레임의 차이를 설명해 주세요.

답변: 가상메모리를 일정한 크기의 블록으로 나누어 관리하는 방식. 페이지는 가상 주소공간을 나눈 단위, 프레임은 물리 메모리를 나눈 단위. 페이지와 프레임의 크기는 같음. 운영체제는 페이지 테이블을 이용해 각 가상 페이지가 어느 물리 프레임에 저장돼있는지 확인. 연속된 가상 주소 페이지들이 물리메모리에서는 서로 떨어진 프레임에 저장될 수도 있음
 
참고 자료:

## [COMMON-048] Page Fault가 발생했을 때 처리 과정을 설명해 주세요.

답변: 프로세스가 접근하려는 페이지가 현재 물리 메모리에 존재하지 않을 때 발생. 먼저 cpu가 해당 페이지에 접근하면 page fault 예외 발생 -> 운영체제로 제어권 넘어감. 운영체제는 접근이 유효한지 확인 -> 잘못됐으면 프로세스 종료, 유효하다면 디스크 스왑 영역이나 실행파일에서 필요한 페이지 찾음. 마지막으로 페이지테이블 갱신 -> 중단 명령어 재실행

## [COMMON-049] TLB가 무엇이고, TLB를 사용하면 왜 주소 변환이 빨라지는지 설명해 주세요.

답변: 최근 사용한 가상페이지 번호, 물리 프레임 번호의 매핑 정보를 저장하는 고속 캐시 / 가상주소를 물리 주소로 변환할 때 메모리에 있는 페이지 테이블을 조회하는데, TLB에 필요한 변환정보가 존재하면 페이지테이블을 메모리에서 다시 읽지 않고 바로 물리주소를 구할 수 있음
