## [COMMON-042] Code, Data, Heap, Stack 영역의 역할을 설명해 주세요.

답변:
Code - 프로그램의 기계어 코드가 저장 / ReadOnly

## [COMMON-048] Page Fault가 발생했을 때 처리 과정을 설명해 주세요.

답변:
Page Falut Interrupt가 발생하면 운영체제가 올바른 메모리 영역인지 확인한 뒤(유효성 검사), 물리 메모리에 빈 공간(프레임)이 있는지 확인한 뒤 공간이 없으면 메모리에 있는 기존 페이지를 보조 기억 장치로 이동시킨 뒤 보조 기억 장치에 있는 최초 요구했던 페이지를 물리 메모리로 가져온다. (페이지 교체)
이후, 페이지 테이블을 업데이트하고 인터럽트를 종료한다.

## [COMMON-049] TLB가 무엇이고, TLB를 사용하면 왜 주소 변환이 빨라지는지 설명해 주세요.

답변:
Translation Lookaside Buffer의 약어이며, 운영체제의 가상 주소를 물리 주소로 변환하는데에 사용된다.
페이지 테이블 또는 세그먼트 테이블의 가상주소와 매핑되는 물리 주소를 저장한다.
가상 주소를 찾을때 페이지 테이블은 프레임을 찾고, 페이지 오프셋 비트를 이용하여 실제 주소를 알아낸다.
다만, 이때 페이지 테이블을 메모리에 저장할 경우 메모리를 읽는 시간이 물리 주소를 이용하여 접근했을때에 비해 2배로 늘어나게 되지만, TLB를 두게되면 시간을 줄일 수 있게 되는 것이다.

## [COMMON-050] 내부 단편화와 외부 단편화의 차이에 대해 설명해 주세요.

답변:
내부 단편화는 필요 없는 자료가 할당 영역 내에 있지만 쓰이지 않는 현상을 이야기하고, 페이징 방식을 사용할때 발생할 ㅅ ㅜ있는 문제이다.
외부 단편화는 여유 공간이 여러 조각으로 나뉘는 현상이다. 여유 공간을 사용할수는 있지만, 조각이 너무 작게 나 있어 I/O 성능을 저하시키는 문제가 발생한다.

## [COMMON-056] Deadlock을 예방하거나 회피하는 방법에 대해 설명해 주세요.

답변: 
데드락을 처리하기 위해서는 예방, 회피, 탐지 및 복구 방식이 존재한다.

회피는 banker, wait-die, wound-wait 기법 등을 통하여 운영체제가 요청을 직접 분석하여 데드락이 발생할 가능성을 확인하는 방식이다.

## [COMMON-058] Hash Table이 무엇이고, 해시 충돌을 해결하는 방법을 설명해 주세요.

답변:
Key-value 형태의 데이터를 저장하는 자료구조이며 삽입 조회 삭제가 O(1) 시간 복잡도를 가진다.
해시 충돌은 서로 다른 Key가 동일한 인덱스로 변환되는 현상이며, 이를 완전히 피할 수는 없다.
동일 인덱스에 여러 데이터를 Linked List와 같은 형태로 저장하는 Chaining 방식이나
충돌이 발생했을 때 다른 빈 슬롯을 탐색하여 저장하는 Open Addressing 방식이 대표적인 해결책이다.

## [COMMON-059] Stack, Queue, Deque의 차이와 사용 사례를 설명해 주세요.

답변:
Stack은 Last In, First Out이며 push, pop, peek 등의 기능이 존재한다. 함수 call stack, 괄호 검사등에 사용 된다.
Queue는 First In, First Out이며 enqueue, dequeue, front 등의 기능이 존재한다. 작업 스케줄링, 메세지 큐등에 사용된다.
Deque는 Double Ended Queue의 줄임말이며 이름에서 알 수 있다시피 양쪽 끝에서 삽입과 삭제가 가능한 큐이다. 슬라이딩 윈도우 문제나 회문 검사등에 사용된다. 또한 Stack과 Queue 기능이 동시에 필요한 경우에도 사용된다.

## [COMMON-060] Tree, Binary Tree, Binary Search Tree의 차이에 대해 설명해 주세요.

답변:
트리는 계층적 자료구조로, 부모-자식 관계로 노드들이 연결되며 Cycle이 존재하지 않는 구조를 말한다.
이진 트리는 트리의 한 종류로, 각 노드가 최대 2개의 자식을 가질 수 있는 구조이다.
BST는 이진 트리의 한 종류로, 왼쪽 서브 트리의 모든 값은 부모 노드의 값보다 작으며 오른쪽 서브 트리의 모든 값은 부모 노드보다 큰 정렬 규칙을 가지는 구조이다.
