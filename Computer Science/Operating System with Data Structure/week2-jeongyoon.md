## [COMMON-042] Code, Data, Heap, Stack 영역의 역할을 설명해 주세요.

답변: 스택 영역은 함수의 호출과 관계되는 지역 변수와 매개변수가 저장되는 영역입니다. 힙 영역과는 반대로 메모리의 높은 주소에서 낮은 주소의 방향으로 할당되고 재귀 함수가 너무 깊게 호출되거나 함수가 지역변수를 너무 많이 가지고 있어 스택 영역을 초과하면 stack overflow 에러가 발생하게 되는 것입니다.

힙 영역은 런타임의 크기가 결정되는 영역입니다. 사용자에 의해 공간이 동적으로 할당 및 해제되며 주로 참조형 데이터 등의 데이터가 할당되고 메모리의 낮은 주소에서 높은 주소의 방향으로 할당됩니다.

데이터 영역은 전역 변수나 static 변수 등 프로그램이 사용할 수 있는 데이터를 저장하는 영역입니다. 어떤 프로그램에서 전역 / static 변수를 참조하는 코드가 존재한다면 이 프로그램은 컴파일 된 후, data 영역을 참조하게 됩니다. 프로그램의 시작과 함께 할당되며 프로그램이 종료되면 소멸합니다.

텍스트 영역은 프로그램이 실행될 수 있도록 CPU가 해석 가능한 기계어 코드가 저장되어 있는 공간으로 프로그램이 수정이 되면 안 되기 때문에 ReadOnly 상태로 저장되어 있습니다.

## [COMMON-046] 가상 메모리가 무엇이고, 왜 필요한지 설명해 주세요.

답변: 가상 메모리란 물리 메모리의 크기에 구애받지 않고 각 프로세스에게 실제보다 훨씬 크고 연속된 것처럼 보이는 메모리 공간을 할당하는 기술입니다. 

현대 운영체제가 멀티태스킹을 원활하게 수행할 수 있는 것은 모두 가상 메모리가 있었기 때문입니다. 가상 메모리는 실제 RAM 용량보다 훨씬 큰 프로그램을 실행할 수 있게 해주어 물리 메모리의 한계를 극복합니다.

또한, 프로그램의 모든 부분을 RAM에 올리는 대신 지금 당장 필요한 부분만 먼저 올리고 나머지는 디스크 영역에 보관합니다. 덕분에 한정된 RAM으로 더 많은 프로세스를 동시에 실행 할 수 있어 멀티 태스킹 효율을 증대시킬 수 있습니다.

각 프로세스는 자신만의 가상 주소 공간에 완벽히 격리되는데 이로 인해 한 프로세스의 오류가 다른 프로세스의 메모리를 침범하는 것을 막을 수 있어 메모리를 보호할 수 있습니다.

새로운 프로세스를 만들 때, 부모 프로세스의 메모리 전체를 복사하는 대신 페이지 테이블이라는 주소록만 복사합니다. 이후 데이터가 변경될 때 해당 부분만 복사하기 때문에 프로세스의 생성 속도를 크게 향상시켜줍니다.

## [COMMON-048] Page Fault가 발생했을 때 처리 과정을 설명해 주세요.

## [COMMON-049] TLB가 무엇이고, TLB를 사용하면 왜 주소 변환이 빨라지는지 설명해 주세요.

답변: TLB란 가상 주소를 물리 주소로 빠르게 변환하기 위해서 사용하는 캐시 메모리입니다.

가상 메모리 시스템에서는 CPU가 사용하는 주소가 모두 가상 주소입니다. 이를 실제 물리 메모리 주소로 변환하기 위해 페이지 테이블을 사용합니다. 하지만 페이지 테이블은 메인 메모리에 위치하므로 주소를 변환할 때마다 메모리에 접근하는 것은 성능 저하를 일으킵니다.

따라서 TLB는 페이지 테이블의 일부분을 캐시 형태로 보관합니다. 최근에 변환한 가상 주소 ↔ 물리 주소 매핑을 저장함으로서 페이지 테이블에 접근하지 않고서도 주소를 쉽게 변환할 수 있습니다.

데드락은 한 번에 하나의 프로세스만 자원을 사용할 수 있어야 하는 상호 배제, 이미 자원을 가진 상태에서 다른 자원을 추가로 기다리는 상태인 점유와 대기, 강제로 자원을 뺏을 수 없는 비선점, 프로세스들이 원형으로 서로의 자원을 기다리는 상태인 순환 대기 네 가지 조건으로 발생합니다.

자원을 동시에 사용할 수 있도록 만들어 상호 배제를 제거하거나, 자원을 가진 상태에서 다른 자원을 요청하지 못하게 hold and wait를 제거하거나, 자원을 강제로 회수해 비선점을 제거합니다. 또는 자원 요청 순서를 정해 원형 대기를 막습니다.

## [COMMON-058] Hash Table이 무엇이고, 해시 충돌을 해결하는 방법을 설명해 주세요.

답변: 해시테이블은 Key-Value 형태로 저장되는 자료구조로 해시 함수를 이용해 데이터를 저장할 위치를 계산하는 자료구조입니다. 평균적으로 O(1)의 시간 복잡도로 데이터를 검색, 삽입, 삭제할 수 있습니다.

두 개의 Key가 같은 위치를 가지는 상황 같은 현상을 해시 충돌이라고 합니다. 해시 충돌을 해결하기 위한 방법으로는 체이닝과 개방주소법이 있습니다. 

체이닝은 같은 주소로 해싱될 때 연결 리스트로 연결하는 방식입니다. 개방 주소법은 추가 공간을 활용하지 않고 충돌을 해결하고자 하여 충돌이 일어났을 때, 충돌 주소의 바로 다음 위치에 저장해버립니다. 빈 bucket이 있을 때까지 다음 위치를 탐색하게 됩니다.

## [COMMON-059] Stack, Queue, Deque의 차이와 사용 사례를 설명해 주세요.

답변: stack은 후입선출(LIFO) 구조로 마지막에 들어온 데이터가 먼저 나가는 자료구조입니다. 함수 호출 스택, Undo 기능 등에 사용됩니다. Queue는 선입선출(FIFO) 구조로 먼저 들어온 데이터가 먼저 처리되며 작업 스케줄링, BFS 등에 활용됩니다. Deque는 양쪽 끝에서 삽입과 삭제가 가능한 자료구조로 Stack과 Queue를 모두 구현할 수 있으며 슬라이딩 윈도우 알고리즘처럼 양방향 처리가 필요한 경우에 적합합니다.

## [COMMON-060] Tree, Binary Tree, Binary Search Tree의 차이에 대해 설명해 주세요.

답변: Tree는 부모-자식 관계로 이루어진 계층형 비선형 자료구조로 자식의 개수에는 제한이 없습니다. Binary Tree는 각 노드가 최대 두 개의 자식만 가질 수 있는 트리이며 값의 크기에 대한 규칙은 없습니다. 

Binary Search Tree(BST)는 Binary Tree의 한 종류로 모든 노드에서 왼쪽 서브트리의 값은 부모보다 작고 오른쪽 서브트리의 값은 부모보다 큰 정렬 규칙을 만족합니다. 이 규칙 덕분에 평균적으로 검색, 삽입, 삭제를 O(log n)에 수행할 수 있지만, 트리가 한 쪽으로 치우치면 O(n)까지 성능이 저하될 수 있어 균형 이진 탐색 트리를 사용하기도 합니다.
