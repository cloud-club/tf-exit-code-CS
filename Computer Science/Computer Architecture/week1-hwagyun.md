## [COMMON-001] CPU의 주요 구성 요소인 ALU, 제어장치, 레지스터에 대해 설명해 주세요.

답변: 
레지스터는 CPU 내에서 연산 시 필요한 데이터들에 빠르게 접근할 수 있도록 데이터를 저장하는 역할을 담당합니다.
ALU는 산술 및 논리연산을 담당하는 역할로, 덧셈/곱셈과 함께 AND/OR 등의 논리연산까지 담당합니다.
제어장치는 CPU 내부 구성요소들의 동작 순서를 조정하는 장치로서, 명령어를 해독해 ALU를 통한 연산을 어떻게 수행할 지, 어느 레지스터에서 데이터를 가져올 지 등에 대한 실제 동작신호를 제어합니다.

답변(추가):
레지스터는 범용으로 사용되는 레지스터와 특수한 목적으로 사용되는 레지스터가 있습니다.
- 프로그램 카운터 (PC): CPU가 다음에 가져오거나 실행할 명령어의 메모리 상 주소를 저장합니다.
- 명령어 레지스터 (IR): CPU가 현재 실행하는 명령어가 저장되는 위치입니다. PC에 있던 주소로부터 명령어를 가져와 IR에 저장하며 PC값이 다음 명령어의 주소로 변경됩니다.
- 스택 포인터 (SP): 프로그램 실행 간 사용되는 메모리스택의 최상단, 또는 스택 경계를 가리키는 메모리 주소를 저장합니다. 프로그램 실행 간 함수호출 시, 각 함수에서 사용되는 지역변수, 함수인자, 반환주소, 저장 레지스터 값들이 저장/삭제될 때 본 레지스터의 값이 변경됩니다.

## [COMMON-002] 명령어 사이클이 무엇인지 Fetch, Decode, Execute 단계로 설명해 주세요.

답변:
명령어 사이클이란 CPU가 프로그램의 명령어를 처리하기 위해 동작하는 방식을 의미하는 것으로, 크게 Fetch -> Decode -> Execute, 경우에 따라 Store까지의 과정을 의미합니다.
Fetch는 CPU가 PC 레지스터가 가리키는 메모리 주소로부터 명령어를 가져오는 것을 의미합니다.
Decode는 인출된 명령어를 이해하고 실행할 수 있도록 해독하는 것을 의미합니다.
Execute는 해독된 명령어를 실제로 실행하는 것을 의미하는데, 크게 ALU 연산, 메모리접근, 분기, 결과저장 등을 수행합니다.

## [COMMON-003] 프로그램이 실행되기까지의 과정을 컴파일, 링킹, 로딩, 실행 흐름으로 설명해 주세요.

답변:
프로그램은 컴파일, 링킹을 통해 실행 가능한 프로그램의 형태가 되고, 사용자가 희망하는 시점에 메모리에 로딩되어 최종적으로 실행됩니다.

컴파일은 개발된 원시코드를 컴퓨터가 이해할 수 있는 기계어로 번역하는 과정을 의미합니다. 해당 작업을 통해 어셈블리 코드를 거쳐 목적파일 (Object File)이 생성됩니다.
이어 여러개의 목적파일이 필요한 라이브러리를 결합하고, 목적파일 간 함수/변수참조 문제 해결 등을 통해 실제 실행파일을 만드는 링킹과정이 진행됩니다. 이렇게 만들어진 파일은 Windows에서는 주로 PE 형식의 .exe, Linux에서는 ELF 형식을 사용합니다.

이후 사용자가 프로그램 실행을 희망하여 클릭이나 명령어를 통해 로딩명령이 전달되면, OS에 의해 메모리가 준비된 후 프로그램에 연결됩니다.
완전히 준비가 되면 OS는 실행 파일의 진입점으로 제어를 넘기게 되며, 해당 상태를 일반적으로 '프로그램이 실행되었다'라고 표현합니다.
